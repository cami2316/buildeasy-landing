Pixel & Analytics – Guia da Landing Page

Esta página é só para você / time interno. Aqui ficam os códigos e a lista de eventos para conectar: Google Analytics 4 Meta Pixel Microsoft Clarity

1. Google Analytics 4 (GA4)

Objetivo: ver visitas, países, aparelhos, páginas vistas.

  1. Crie uma conta em Google Analytics (GA4).
  2. Crie uma Property para o site cami2316.github.io/buildeasy-landing.
  3. Copie o Measurement ID, algo como G-XXXXXXXXXX.

Depois, cole este código dentro do <head> da página que quiser rastrear (ex: index.html):

<!-- Google Analytics 4 -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>

Trocar G-XXXXXXXXXX pelo seu ID real.

Eventos recomendados (landing)

  • waitlist_cta_click – clique em “Entrar na lista de espera”.
  • waitlist_page_view – quando a pessoa abre waitlist.html.
  • ai_page_view – quando visita ia.html.
  • ocr_page_view – quando visita ocr.html.
  • pricing_page_view – quando visita pricing.html.

Exemplo de função para disparar clique do CTA (colocar perto do final do <body>):

<script>
  function trackWaitlistClick() {
    if (window.gtag) {
      gtag('event', 'waitlist_cta_click', {
        event_category: 'engagement',
        event_label: 'Hero button'
      });
    }
    if (window.fbq) {
      fbq('trackCustom', 'WaitlistCTAClick');
    }
    if (window.clarity) {
      clarity('event', 'waitlist_cta_click');
    }
  }
</script>

E no botão da landing (index.html):

<a class="btn btn-primary" href="waitlist.html" onclick="trackWaitlistClick()">
  Entrar na lista de espera
</a>

2. Meta Pixel (Facebook / Instagram)

Objetivo: rodar anúncios e fazer remarketing.

  1. Abra o Meta Business (Business Manager).
  2. Crie um Pixel novo.
  3. Copie o Pixel ID, algo como 123456789012345.

Código para colar logo depois do GA4, ainda dentro do <head>:

<!-- Meta Pixel -->
<script>
  !function(f,b,e,v,n,t,s)
  {if(f.fbq)return;n=f.fbq=function(){n.callMethod?
  n.callMethod.apply(n,arguments):n.queue.push(arguments)};
  if(!f._fbq)f._fbq=n;n.push=n;n.loaded=!0;n.version='2.0';
  n.queue=[];t=b.createElement(e);t.async=!0;
  t.src=v;s=b.getElementsByTagName(e)[0];
  s.parentNode.insertBefore(t,s)}(window, document,'script',
  'https://connect.facebook.net/en_US/fbevents.js');
  fbq('init', 'YOUR_PIXEL_ID');
  fbq('track', 'PageView');
</script>
<noscript>
  <img height="1" width="1" style="display:none"
       src="https://www.facebook.com/tr?id=YOUR_PIXEL_ID&ev=PageView&noscript=1"/>
</noscript>

Trocar YOUR_PIXEL_ID pelo seu ID real.

Eventos personalizados sugeridos

  • WaitlistCTAClick – clique na lista de espera.
  • AIPageView – visita página da IA.
  • OCRPageView – visita página do OCR.

3. Microsoft Clarity (mapa de calor)

Objetivo: ver mapa de calor, gravações de sessão, onde o usuário emperra.

  1. Crie conta em clarity.microsoft.com.
  2. Crie um projeto para cami2316.github.io/buildeasy-landing.
  3. Copie o ID do Clarity, algo como a1b2c3d4e5.

Código para colar no <head>, depois do Pixel:

<!-- Microsoft Clarity -->
<script type="text/javascript">
  (function(c,l,a,r,i,t,y){
      c[a]=c[a]||function(){(c[a].q=c[a].q||[]).push(arguments)};
      t=l.createElement(r);t.async=1;t.src="https://www.clarity.ms/tag/"+i;
      y=l.getElementsByTagName(r)[0];y.parentNode.insertBefore(t,y);
  })(window, document, "clarity", "script", "CLARITY_ID");
</script>

Trocar CLARITY_ID pelo ID real.

4. Eventos futuros dentro do app (BuildEasy)

Quando o app estiver rodando, estes são eventos bons para enviar para GA4 / Pixel:

  • estimate_created – usuário criou um estimate.
  • estimate_converted_to_invoice – converteu para invoice.
  • work_created – nova obra criada.
  • work_profit_viewed – abriu tela de lucro por obra.
  • daily_work_created – registrou diária.
  • receipt_scanned_ocr – usou OCR para recibo.
  • ai_suggestion_applied – aceitou sugestão da IA.
  • waitlist_signup / plan_upgrade – upgrade de plano.

Quando você tiver os IDs reais (GA, Pixel, Clarity), basta: 1) copiar os blocos acima para o <head> do index.html e waitlist.html 2) ativar a função trackWaitlistClick() no botão da landing.
